# 📊 Mobile Sales Dashboard (Motorola Case Study)

# 📌 Overview

This project is an interactive **Mobile Sales Dashboard** built using Power BI.
The analysis is performed on **Motorola mobile sales data** as a case study to understand sales performance, customer behavior, and business trends.

# 🔧 Tools & Technologies Used

* Power BI
* Microsoft Excel

# 📊 Key Metrics (KPIs)

* **Total Sales:** 308M
* **Total Quantity Sold:** 8K
* **Total Transactions:** 2K
* **Average Sales Value:** 41K

# 📈 Dashboard Features

# 🌍 City-wise Sales Analysis

* Map visualization showing sales distribution across major cities
* Helps identify high-performing locations

# 📅 Monthly Sales Trend

* Line chart showing monthly sales/quantity trends
* Highlights seasonal fluctuations

# ⭐ Customer Ratings

* Funnel chart representing customer rating distribution (1–5)
* Shows overall customer satisfaction

# 💳 Payment Method Analysis

* Pie chart of payment modes:

  * Credit Card
  * UPI
  * Debit Card
  * Cash
* Identifies most preferred payment method

# 📱 Product / Model Analysis

* Comparison of different mobile models
* Helps understand which products perform better

# 📆 Sales by Day

* Analysis of sales across days of the week
* Identifies peak sales days

# 🏷️ Brand Context

* This project focuses specifically on **Motorola** sales data as a case study within the mobile domain

# 🎯 Key Insights

* Certain cities contribute significantly to overall sales
* Credit Card is the most used payment method
* Sales show monthly variations and trends
* Majority of customers give high ratings (4–5)
  
# 🚀 Conclusion
This dashboard demonstrates how sales data can be transformed into meaningful insights using Power BI.
It highlights skills in **data visualization, business analysis, and dashboard development**.
